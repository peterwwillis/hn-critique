Main Points & Arguments

  • String concatenation in loops causes O(n²) copying due to String immutability; using StringBuilder improves performance.
  • Using streams inside loops can cause accidental O(n²) complexity; replacing with single-pass aggregation reduces CPU usage.
  • String.format() is slow in hot paths because it parses the format string every call; prefer StringBuilder or partial formatting.
  • Autoboxing in tight loops creates many temporary wrapper objects, increasing heap usage and GC pressure; use primitives instead.
  • Using exceptions for control flow is costly due to stack trace generation; pre-validate inputs to avoid frequent exceptions.
  • Broad synchronization locks cause contention bottlenecks; use ConcurrentHashMap and LongAdder for high concurrency.
  • Repeated creation of expensive reusable objects like ObjectMapper or DateTimeFormatter hurts performance; reuse instances.
  • Virtual thread pinning in JDK 21–23 occurs when synchronized blocks with blocking I/O prevent unmounting carrier threads; use ReentrantLock or upgrade to JDK 24 to mitigate.
