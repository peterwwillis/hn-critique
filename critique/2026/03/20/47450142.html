Summary

The Trivy ecosystem experienced a critical supply chain attack on March 19, 2026, where a threat actor used compromised credentials to publish malicious releases and force-push malicious commits to several Trivy-related GitHub Actions repositories. The attack involved injecting credential-stealing malware into multiple components, affecting various versions and distribution channels, but was mitigated by removing malicious releases and recommending updates to safe versions.

Main Points & Arguments

  • On March 19, 2026, a threat actor published a malicious Trivy v0.69.4 release and force-pushed malicious commits to aquasecurity/trivy-action and aquasecurity/setup-trivy repositories.
  • The attack was a continuation of a supply chain compromise that began in late February 2026, exploiting incomplete credential rotation.
  • Malicious code injected included an infostealer that extracted secrets from runner environments and exfiltrated them to attacker-controlled infrastructure.
  • The compromised releases were distributed via multiple channels including GHCR, Docker Hub, and official package repositories.
  • Users pinned to safe versions (Trivy v0.69.3 or earlier, trivy-action v0.35.0, setup-trivy v0.2.6) or using immutable releases were not affected.
  • The advisory recommends immediate rotation of exposed secrets, auditing workflows and artifacts, and pinning GitHub Actions to immutable commit SHAs.
  • Verification methods using sigstore and cosign are provided to confirm the integrity of Trivy binaries and container images.
