Summary

The article argues that Wayland, a display server protocol intended to replace X11 on Linux desktops, has hindered Linux desktop progress over 17 years due to slow adoption, incomplete features, and user-facing issues. It critiques Wayland's security model, performance claims, fragmentation, and the forced migration from X11, concluding that Wayland is not yet ready for widespread use.

Main Points & Arguments

  • Wayland was created to replace the aging X11 protocol with a simpler, more modern display server.
  • After 17 years, Wayland adoption is around 40-60%, which the author considers slow compared to other projects like PipeWire.
  • Wayland's security model restricts application interactions in ways that frustrate users and break workflows.
  • Performance improvements with Wayland over X11 are inconsistent and sometimes worse, especially with Nvidia hardware.
  • Wayland is a protocol, not a single implementation, leading to fragmentation and lack of standardized features like drag-and-drop and screen sharing.
  • Many common utilities and legacy software are not fully supported or broken under Wayland, causing user frustration.
  • Developers express frustration at user complaints, but users are forced to switch despite ongoing issues.
  • The author predicts a future where Wayland support declines, a new protocol emerges, and fragmentation continues.
