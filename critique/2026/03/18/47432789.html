Summary

The article discusses techniques to reduce the size of Haskell binaries, which can be quite large due to many transitive dependencies. It explores using linker options like -split-sections, --gc-sections, and identical code folding (ICF) with the lld linker to significantly reduce binary size, using the pandoc binary as a test case.

Main Points & Arguments

  • Haskell binaries can be very large, often around 100MB or more.
  • Using GHC options like -split-sections and linker options like --gc-sections can reduce binary size by removing dead code.
  • The lld linker supports identical code folding (ICF), which merges functionally equivalent sections to further reduce binary size.
  • Applying these techniques to the pandoc binary reduced its size from 113MB to 64MB.
  • ICF works well because Haskell binaries often contain many duplicated code sections due to inlining and specialization.
  • Debugging with these optimizations can be complicated because some debug info sections may be folded or removed.
  • The author experimented with tools like bloaty and kcov for binary analysis but found them incompatible with Haskell binaries.
