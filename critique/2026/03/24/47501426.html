Summary

The article reports a critical security issue in the litellm Python package version 1.82.8 available on PyPI, which contains a malicious .pth file that automatically executes a credential-stealing script on Python interpreter startup. This supply chain attack collects sensitive data including environment variables, SSH keys, cloud credentials, and more, then encrypts and exfiltrates it to an attacker-controlled server.

Main Points & Arguments

  • The litellm==1.82.8 PyPI package includes a malicious litellm_init.pth file that runs automatically without import.
  • The malicious script collects extensive sensitive information such as system info, environment variables, SSH keys, cloud credentials, crypto wallets, and CI/CD secrets.
  • Collected data is encrypted using AES-256 and RSA and exfiltrated to https://models.litellm.cloud/, a domain different from the official litellm.ai.
  • This compromise affects local machines, CI/CD pipelines, Docker containers, and production servers where litellm 1.82.8 was installed.
  • Users are advised to check for the malicious file, rotate all credentials, and PyPI is urged to remove the compromised package version immediately.
