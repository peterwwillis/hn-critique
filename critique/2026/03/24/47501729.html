LiteLLM Python package compromised by supply-chain attack (github.com)

Summary

The article reports a critical supply-chain attack on the LiteLLM Python package version 1.82.8, where a malicious .pth file was included in the PyPI package. This file automatically executes a credential-stealing script upon Python interpreter startup, exfiltrating sensitive data such as environment variables, SSH keys, cloud credentials, and other secrets to an attacker-controlled server.

Main Points & Arguments

  • The LiteLLM 1.82.8 PyPI package contains a malicious litellm_init.pth file that executes automatically on Python startup without import.
  • The malicious script collects extensive sensitive data including system info, environment variables, SSH keys, cloud credentials, crypto wallets, and CI/CD secrets.
  • Collected data is encrypted with AES-256 and RSA and exfiltrated to https://models.litellm.cloud/, a domain distinct from the official LiteLLM domain.
  • The attack affects local development machines, CI/CD pipelines, Docker containers, and production servers where the compromised package was installed.
  • The compromised version has been confirmed as 1.82.8; other versions have not yet been checked but may also be affected.
  • Recommended actions include immediate removal of version 1.82.8 from PyPI, checking for the malicious .pth file, and rotating all potentially compromised credentials.
