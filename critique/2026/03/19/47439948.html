Summary

The article discusses a critical bug in PSpice's AES-256 encryption implementation that drastically reduces the effective keyspace, enabling a brute-force attack to recover user keys in seconds. The author introduces SpiceCrypt, a tool that exploits this vulnerability to decrypt PSpice-encrypted SPICE model files, restoring interoperability with other simulators.

Main Points & Arguments

  • PSpice uses a proprietary encryption scheme with six modes; mode 4 incorporates user-supplied keys but has a key derivation bug.
  • The bug causes only a 4-byte portion of the key to be used in AES-256 encryption, reducing the keyspace from 2^256 to 2^32.
  • This reduction enables a brute-force attack using a known plaintext prefix to recover the user key quickly.
  • The root cause is likely a copy-paste error where the short key intended for DES was used for AES instead of the extended key.
  • SpiceCrypt is a released tool that implements this attack and supports decryption of multiple SPICE encryption formats.
  • Fixing the bug now would break compatibility with encrypted models created over the past twelve years.
