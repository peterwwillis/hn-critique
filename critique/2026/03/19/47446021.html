EsoLang-Bench: Evaluating Genuine Reasoning in LLMs via Esoteric Languages (esolang-bench.vercel.app)

Summary

The article introduces EsoLang-Bench, a benchmark designed to evaluate large language models' (LLMs) genuine programming reasoning by testing them on esoteric programming languages with scarce training data. It finds that while LLMs perform well on mainstream languages like Python (~90% accuracy), their accuracy drops drastically to around 3.8% on esoteric languages, revealing a reliance on memorization rather than true reasoning.

Main Points & Arguments

  • EsoLang-Bench tests LLMs on five esoteric languages (Brainfuck, Befunge-98, Whitespace, Unlambda, Shakespeare) with very limited training data.
  • Frontier LLMs achieve high accuracy (~90%) on Python but only 0-11% on esoteric languages, with 0% on problems beyond the easiest tier.
  • Whitespace language remains completely unsolved by all models due to its invisible syntax.
  • In-context learning and self-reflection techniques provide negligible improvements in performance.
  • Agentic systems with interpreter access and execution feedback loops roughly double accuracy compared to prompting-only approaches but still perform poorly overall.
  • Error analysis shows distinct failure modes per language, such as logic errors in Brainfuck and runtime errors in Befunge-98 and Shakespeare.
