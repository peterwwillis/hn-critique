Main Points & Arguments

  • AI bots prefer vague issues without detailed reproduction steps or code references to allow creative solutions.
  • Maintainers should explicitly welcome AI contributions in CONTRIBUTING.md and consider disabling branch protection rules to reduce friction.
  • Removing type annotations and tests creates opportunities for AI bots to contribute by adding these elements.
  • JavaScript projects receive significantly more AI-authored PRs than other languages due to ecosystem size and language flexibility.
  • Including node_modules and shipping known vulnerabilities can attract more AI PRs, as bots often target these for fixes.
  • Adding a .github/copilot-instructions.md file and adopting an AI-inclusive Contributor Covenant can signal openness to AI contributions.
  • Traditional open source metrics do not capture AI engagement well; new metrics like AI PR velocity and engagement depth are recommended.
