Main Points & Arguments

  • The Xbox 360 CPU is a three-core PowerPC chip with a 1-MB L2 cache shared among cores.
  • A new instruction, xdcbt, was added to prefetch data directly into L1 cache, bypassing L2 cache to improve performance but breaking memory coherency guarantees.
  • Using xdcbt incorrectly caused heap corruption crashes due to incoherent views of memory across cores.
  • The bug was initially thought to be caused by explicit use of xdcbt, but later found to also occur due to speculative execution of xdcbt instructions by the branch predictor.
  • Speculative execution of xdcbt could not be canceled, causing side effects identical to real execution, leading to crashes even when the instruction was not actually executed.
  • The bug was ultimately deemed too dangerous to allow xdcbt instructions in game code, and games shipped without using it.
