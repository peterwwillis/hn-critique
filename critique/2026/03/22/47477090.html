Summary

The article discusses the use of curried functions in functional programming, comparing them to parameter list and tuple styles. It argues that while currying offers elegant partial application and a pleasing theoretical structure, it has drawbacks such as performance overhead, awkward type shapes, and difficulties in composition, suggesting that tuple style functions can be a practical alternative.

Main Points & Arguments

  • Curried functions are common in purely functional languages and allow partial application naturally.
  • Partial application can also be achieved with parameter list or tuple style functions using syntactic sugar.
  • Curried functions create intermediate closures which may impact performance, though optimizers can mitigate this.
  • Curried function types have an asymmetrical shape that complicates composition and type unification in some contexts.
  • Tuple style functions align better with the conceptual input-output function model and can simplify certain programming tasks.
  • Dependent function types in languages like Coq or Agda benefit from currying due to dependent typing on input values.
  • Despite its elegance, currying is not always the best choice, and tuple style functions deserve consideration, especially in language or library design.
