stainlu
The debate here is framed as MCP-vs-CLI but the real answer is both, for different things. MCP gives you a uniform interface for tool discovery and composition across agents -- one protocol, any tool, swap implementations without changing the agent. Direct CLI tools are faster and cheaper per call but each requires its own integration.

Browser DevTools falls squarely in the "direct access is better" category. A single DOM snapshot can burn thousands of tokens through MCP's protocol envelope, and high-frequency interactions (stepping through network requests, inspecting computed styles) compound that cost fast. paulirish's standalone CLI is the pragmatic answer: provide a direct path for the hot loop.

Where MCP still wins is the cold path -- tool discovery, one-off integrations, composing tools from different vendors without writing glue code. The mistake is treating this as an either/or when it's really about matching the protocol overhead to the interaction frequency. Hot paths get direct access, cold paths get the standard.
