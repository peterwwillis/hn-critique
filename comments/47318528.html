Secrets management for AI agents is one of those problems that looks simple until you think about the threat model carefully. The core tension: agents need credentials to be useful, but agents are also susceptible to prompt injection and can be manipulated into exfiltrating those credentials.

The traditional approach of injecting secrets as environment variables works for CI/CD pipelines because the pipeline is deterministic — it runs a fixed script. AI agents are not deterministic. They interpret instructions, fetch external content, and make decisions. An agent that has database credentials and also fetches a webpage that contains a prompt injection is a fundamentally different threat profile than a CI job.

What I think the industry needs to converge on: scoped, short-lived credentials with audit logging. Give the agent a token that can only access the specific resources it needs for the current task, expires in minutes, and logs every action. If the agent gets manipulated, the blast radius is contained. This is basically the principle of least privilege applied to non-deterministic systems, and it is more important here than anywhere else because the "user" holding the credentials cannot be fully trusted to behave predictably.
